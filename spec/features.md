# 已完成功能清單

## 4.1 認證模組

| 功能 | 行為說明 |
|------|----------|
| 會員登入 | Email + 密碼，取得 JWT Access/Refresh Token 存入 localStorage |
| 會員註冊 | 姓名、Email、密碼（≥8字元）、電話（選填） |
| 自動登入 | App 啟動時有 token → 呼叫 `/users/me` 還原狀態 |
| Token 自動更新 | 401 時自動用 Refresh Token 換新，最多重試一次 |
| 登出 | 清除 localStorage token + context 狀態 |
| 登入守衛 | `requireAuth()` — 未登入時阻擋並開啟登入 Modal |

## 4.2 活動探索模組

| 功能 | 行為說明 |
|------|----------|
| 首頁搜尋 | 關鍵字搜尋標題、地點、標籤、主揪名稱 |
| 地點篩選 | 全台灣或指定縣市（22縣市＋離島） |
| 活動歸屬區分 | **視覺區分**：社團活動顯示「社團認證」標章；個人活動顯示「個人主揪」與主揪頭像 |
| 運動分類篩選 | 支援 8 大核心運動（羽球、排球、籃球等）獨立篩選 |
| 探索標籤 | 首頁快速篩選卡片（7 預設 + 自訂） |
| 活動列表 | 顯示主揪/社團資訊、剩餘名額、運動類別 |

## 4.3 活動詳情與主揪信任

| 功能 | 行為說明 |
|------|----------|
| 活動詳情頁 | 大圖、主揪信任卡（顯示主揪稱號、帶團次數、個人簡介） |
| 報名狀態 | LIMITED：進度條（N/M）；OPEN：參加人數頭像 |
| 報名入口 | 「立即報名」：依 `approvalMode` 提示為「直接加入」或「需審核」 |

## 4.4 報名與審核模組

| 功能 | 行為說明 |
|------|----------|
| 報名流程 | Step1 組別/交通；Step2 聯絡資訊（Line/電話）+ 隱私聲明勾選 |
| 取消限制 | 主揪可自訂「活動前 X 小時禁止取消」 |
| 手動審核 | 主揪可自由選擇報名者通過，**通過時才扣除名額** |
| 自動遞補 | 當有人取消時，系統自動將最早的候補（WAITLISTED）轉為正式（APPROVED）並通知 |
| 標記缺席 | 活動結束後，主揪可標記「未出席」，觸發 XP 懲罰 |

## 4.5 運動稱號系統

| 功能 | 行為說明 |
|------|----------|
| 獨立 XP 計算 | 依運動項目分別累積經驗值（如：羽球 XP、登山 XP） |
| 趣味稱號 | 依等級給予趣味稱號（如：擊球看運氣 → 殺球狂人 → 網前小王子） |
| 稱號顯示 | 顯示於個人頁面、貼文作者旁、以及活動主揪卡片 |
| 經驗發放 | 活動日期過後自動發放；主揪標記缺席則扣除 |

## 4.5 社團模組

| 功能 | 行為說明 |
|------|----------|
| 社團列表 | 社團卡片（Logo、名稱、評分、人數、標籤），骨架屏 |
| 社團頁面 | 三頁籤：社群（貼文）、活動、相簿 |
| 加入社團 | 更新 `joinedClubIds`，Toast 通知 |
| 退出社團 | 更新 `joinedClubIds`，Toast 通知 |
| 活動月曆 | 週/月/年視圖切換日曆 |
| 社團管理 | 編輯名稱/描述/Logo/標籤，管理成員（踢除），刪除社團 |
| 成員等級徽章 | 依參加活動次數顯示等級（見 [data-models.md#clubmemberrank](data-models.md#clubmemberrank)） |

**等級徽章顯示位置**
1. 社團成員列表（管理員 → 管理成員）：名字右側
2. 貼文作者名稱旁：小型 icon
3. 個人資料頁已加入社團卡片（待實作）

## 4.6 社群互動模組

| 功能 | 行為說明 |
|------|----------|
| 發文 | 選擇類型（公告/閒聊/相簿），輸入內容、上傳圖片 |
| 貼文篩選 | 依類型篩選 |
| 貼文排序 | 最新/最早切換 |
| 按讚 | 按讚/取消，即時更新（local state） |
| 巢狀留言 | 頂層 + 回覆（parentId），最多兩層 |
| 編輯/刪除貼文 | 管理員可對任意貼文操作 |
| 刪除留言 | 管理員可刪除 |

## 4.7 個人中心模組

| 功能 | 行為說明 |
|------|----------|
| 個人頁面 | 頭像、名稱、報名活動、加入社團 |
| 取消報名 | 活動卡片上取消，立即移除 |
| 退出社團 | 社團卡片上退出 |
| 編輯資料 | 修改名稱、頭像（PATCH /users/me） |
| 設定 | SettingsModal：Dark Mode、語言（佔位）、登出 |

## 4.8 通知模組

| 功能 | 行為說明 |
|------|----------|
| 通知列表 | 未讀以深色背景標記 |
| 一鍵全讀 | 全部標為已讀 |
| 點擊跳轉 | ACTIVITY → 活動詳情；INVITE/INTERACTION → 社團頁 |
| 未讀徽章 | Nav 鈴鐺顯示未讀數紅點 |

## 4.9 系統功能

| 功能 | 行為說明 |
|------|----------|
| Dark Mode | 全站 Tailwind `dark:` class，localStorage 持久化 |
| Toast | 底部彈出（success/error/info），3 秒消失 |
| 響應式 | 手機底部導覽列 / 桌面浮動 Dock |
| FAB | 浮動建立按鈕 → 選擇活動/社團/貼文 |
| 刪除確認 | 所有破壞性操作用確認 Modal（禁用 `confirm()`） |

## 4.10 體驗與效能優化 (Tier 3)

| 功能 | 行為說明 |
|------|----------|
| 自動滾動加載 | 列表頁（活動、社團、貼文）使用 `IntersectionObserver`，滑到底部自動載入下一頁，無需手動點擊 |
| 搜尋建議 (Autocomplete) | 搜尋列輸入時，即時從 `/activities/suggestions` 取得熱門活動或標籤建議 |
| 全站骨架屏 | 活動列表、社團詳情、個人資料頁、通知列表均實作 Skeleton 預載畫面，減少佈局跳動 |
| 路由原地攔截 | 未登入或權限不足訪問受限頁面時，不強制跳轉，而是顯示「權限鎖定」插畫與引導按鈕 |
| 搜尋防抖 (Debounce) | 所有搜尋輸入框統一 500ms 防抖，降低伺服器負擔 |
